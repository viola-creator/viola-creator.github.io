Core phase (critical path) Background track (non-blocking) Review / sign-off gate
Dependency spine
Nav + design tokens  →  lock first, build once, affects every page
Stay template  →  Stay 1  →  Stays 2 & 3  →  Book Now (date-first funnel)
Workshop availability (Acuity) + Stays availability (PMS, TBC) + Stripe Elements  →  Book Now
Retreats URL decision  →  Artist content  →  product display  →  Artist Story page
FAQ + Access + Contact  →  one IA cluster (shared location / hours / policy)
⚑ Critical path: Nav/tokens → Stay template → Stay 1 → Stays 2/3 → Book Now → Launch. Book Now is the longest pole and highest risk — keep it isolated so the rest can launch even if it lags.

Foundation

Lock the global layer + answer the open decisions so everything else runs in parallel.
  • IA audit

  • Design tokens review
  • Stay template wireframes
  • Book Now spec
  • Artist Story concept

  • Resolve 8 open decisions

1
Nav + Stay template + Stay 1
June wk3 → mid-Jul
New nav live everywhere; first canonical stay page shipped end-to-end (proves the whole loop).

2
Stays 2/3 + Book Now
mid-Jul → mid-Aug
All three stays live; Book Now wired into the date-first funnel. Hardest, highest-risk phase.
  • Stays 2 & 3 content + parallel builds
  • Book Now refine from learnings
  • Merged stays + workshops results
  • Stripe Elements in-panel checkout
  • PMS + Acuity write + reconciliation
  • Full-funnel QA
Gate: founders + Sol (architecture) · Tomomi copy · extra scrutiny on payment/booking sync.  →  Output: 3 stays + Book Now in staging, funnel testable.
3
Artist Story + Info layer + Home
mid-Aug → early Sep
Replace Retreats with Artist Story; consolidate FAQ/Access/Contact; refresh Home.
  • Artist content + product-display decision
  • Artist Story build
  • Retreats 301 / swap + sitemap
  • FAQ + Access + Contact reorg (GEO-aware)
  • Home updates
Gate: founders + area owners per page · Tomomi copy.  →  Output: Artist Story live, info layer reorganized, Home refreshed.
4
Launch prep
Sep · wk 2–3
Polish, QA, performance, clean launch.
  • Cross-browser + mobile
  • Lighthouse every page
  • Broken-link + 404/redirect audit
  • Image optimization
  • Schema validation
  • hreflang check
  • Launch sign-off
  • Prod deploy + 48h monitor
Gate: launch sign-off — Viola, Sol, Tomomi + founders.  →  Output: English site live at maana.jp.
